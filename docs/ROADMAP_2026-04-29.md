## 1. North Star — the product we're actually building

Eavesight gives a roofer a ranked map of every house in their service area, scored across **eight independent lead-quality dimensions**, with one-click access to the homeowner contact, storm history, an estimated job value, and the trigger signals that fired. Every dimension is independently calibrated and explainable. Roofers buy the dimensions that match their business model.

**Why this isn't "another lead-list product":** competitors sell ONE composite score driven by storm + age. We sell **eight composable scores** spanning the actual replacement-trigger taxonomy. A storm-chase roofer pays for `score_storm_damage`. A pre-sale-renovation specialist pays for `score_pre_sale`. Volume roofers pay for the union. Insurance partnerships sell the actuarial layer to carriers. **This unlocks segment-by-segment customer expansion without adding net new data sources** — the underlying signals are mostly in our DB already.

The 22% / 78% storm/non-storm split makes this real: the 78% non-storm replacements decompose into pre-sale (12-18%), age/wear (30-35%), maintenance-evolved (8-12%), insurance non-renewal (5-10%), refinance/HELOC (3-5%), renovation co-occurrence (3-5%), and other minor categories. Each one is a sellable lead segment.

## 7. The 8-dimensional lead score (v1 schema)

This is the headline product structure. All 8 dimensions are normalized to 0-100 scale; missing data renders as `null` (not 0).

| # | Score dimension | What it measures | Primary data source |
|---|---|---|---|
| 1 | `score_storm_damage` | Probability of insurance-covered replacement opportunity | property_storms × hailSize × time-since-event |
| 2 | `score_age_attrition` | Natural-attrition replacement urgency | yearBuilt + 22yr asphalt life cycle |
| 3 | `score_pre_sale` | Pre-listing renovation probability | MLS active/recent listings |
| 4 | `score_recent_purchase` | New-owner renovation propensity | madison_parcel_data.deedDate < 18mo |
| 5 | `score_renovation_active` | Co-occurring renovation triggers | building_permits non-roofing within 12mo |
| 6 | `score_block_momentum` | Neighborhood reroofing wave | reroof_permits within 500m in 12mo |
| 7 | `score_imagery_unchanged` | Prithvi-confirmed roof persistence | inference output: 1 - P(replacement 2011-2023) |
| 8 | `score_insurance_window` | Days remaining on storm-damage claim window | state SOL × storm event date |

**Composite lead score (default rendering):**

```
lead_score = max_of_independent_scores
  weighted by each roofer's preferences (per-roofer adaptive in v3; uniform in v1)
```

Each dimension also exposes:
- `confidence` (0-1)
- `signals_fired[]` (which raw `property_signals` rows contributed)
- `pitch_template` (which roofer-pitch text applies for this dimension)

## 17. Skiptrace pipeline integration (companion: `docs/SKIPTRACE_PIPELINE.md`)

The skiptrace pipeline plan is authored. **Out of v1 scope** — but its Phase 0 (Mock provider) is critical infrastructure that lets v1 demo the full lead-flow end-to-end without burning Tracerfy budget.

### Decisions locked by `SKIPTRACE_PIPELINE.md`

- **All trace lookups go through `SkipTraceProvider` interface in `apps/backend/src/skiptrace/`.** Vendors are interchangeable; swap by config flag.
- **Phase 0 default: `SKIPTRACE_PROVIDER=mock`.** Returns deterministic synthetic data (FCC-reserved 555-0100..555-0199 numbers) so dev/QA/demo flow works without external API calls.
- **Compliance filter (`ComplianceFilter`) wraps every provider.** No provider can return data that bypasses DNC suppression, line-type gating, or litigator suppression. The "if we don't know, we don't show" rule from `project_compliance_policies.md` is enforced architecturally.
- **10 compliance test fixtures** locked: dnc_listed, voip, unknown_line_type, mobile_clean, landline_clean, no_match, ebr_exemption, multi_resident, email_only, litigator_match. All must pass on every CI build.
- **Primary vendor: Trestle** ($0.07/lookup; only vendor with explicit Reseller/VAR clause TOS §3.3.3). Compliance overlay: Twilio Lookup ($0.008/req line-type). Tracerfy ($0.02) and skipreach ($0.05) are TOS-pending.
- **Disqualified vendors (TOS reasons)**: BatchData, PropertyRadar, Endato, TLOxp, LexisNexis Accurint. Don't revisit these.
- **Phased timeline:** Phase 0 mock (weeks 1-2, current) → Phase 1 vendor sandboxes (week 3) → Phase 2 curated pilot (week 4) → Phase 3 internal beta 5-10 roofers (weeks 5-6) → Phase 4 external beta 25-50 roofers (weeks 7-12) → Phase 5 GA.
